# Healthcare Patient Scheduler & Intake Voice Agent

## Overview
This repository contains the architecture and implementation for a production-grade **Healthcare Patient Scheduler & Intake Voice Agent**. The agent acts as an AI receptionist that patients can converse with over the phone to handle routine tasks, significantly reducing the workload for medical front-desk staff.

Instead of relying on static mock data, this project simulates a fully functional hospital environment by integrating with a local Electronic Health Record (EHR) server using FHIR standards and a Retrieval-Augmented Generation (RAG) pipeline for medical triage.

## Core Objectives & Features
1. **Conversational Intake:** Collects preliminary symptoms, duration of illness, and basic patient information.
2. **Automated Scheduling:** Checks clinic calendars and proposes available slots, allowing patients to book, cancel, or reschedule appointments entirely over the phone.
3. **Strict Triage & Guardrails:** Listens for emergency keywords (e.g., chest pain, shortness of breath) and instantly redirects the patient to dial 911 if a medical emergency is detected.
4. **Insurance Verification:** Collects basic insurance provider details for billing context.

## Technology Stack

### Telephony & Speech Processing
*   **Vapi:** Primary engine for handling the phone call lifecycle and bridging the AI to the PSTN network.
*   **Deepgram:** Provides ultra-fast, low-latency Speech-to-Text transcription to accurately parse medical terminology and background noise.
*   **WebRTC:** Serves as a fallback testing environment for browser-to-server audio streaming.

### Data & Backend Integration Layer
*   **FastAPI Mock EHR:** A simulated backend replicating an Epic EHR database. It serves standard FHIR R4 JSON payloads containing realistic (but fake) patient histories and provider directories generated by [Synthea](https://synthetichealth.github.io/synthea/).
*   **Medical Knowledge Base (RAG):** Uses **Supabase Vector / `pgvector`** populated with the `adrianf12/healthcare-qa-dataset` from Hugging Face. This empowers the LLM to accurately answer general triage questions safely while keeping app data and vectors on one managed platform.

## Project Structure
*   `data/`: Contains the seed data and curated RAG knowledge for the project.
    *   `fhir/`: Realistic mock patient and provider JSON bundles from Synthea.
    *   `healthcare_qa.csv`: Legacy tiny medical Q&A dataset kept for reference.
    *   `knowledge_base/`: Primary RAG source for Mercy General policy, department services, FAQ, appointment prep, and safe care routing.
*   `scripts/`: Python utility scripts used to download and set up the seed data (`download_qa_data.py`, `download_synthea_data.py`).
*   `*.md` files: Various planning and architecture documents detailing agent scope, clinic knowledge base, and implementation plans.

## Setup & Installation

1. **Clone the repository and navigate to the directory:**
   ```bash
   cd "Voice  Agent"
   ```

2. **Set up a Python Virtual Environment (Recommended):**
   ```bash
   python -m venv venv
   # On Windows:
   .\venv\Scripts\activate
   # On Mac/Linux:
   source venv/bin/activate
   ```

3. **Install Dependencies:**
   ```bash
   pip install -r requirements.txt
   ```

4. **Acquire Seed Data:**
   Run the setup scripts to download the Synthea FHIR mock data and optional legacy Hugging Face QA dataset:
   ```bash
   python scripts/download_qa_data.py
   python scripts/download_synthea_data.py
   ```
